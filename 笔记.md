## Jetson RAW10 摄像头问题总结

### 一、问题现象
```
OpenCV GStreamer warning: Internal data stream error
reason not-negotiated (-4)
```

### 二、根本原因
**摄像头格式不匹配** - 摄像头是 **RAW10 Bayer格式**（`RG10`），而代码期望的是常见的RGB/YUV格式。

```bash
# 查看摄像头格式
v4l2-ctl --device=/dev/video0 --list-formats-ext

# 输出显示
Pixel Format: 'RG10'  # 10-bit Bayer原始格式
```

### 三、解决方案

#### 1. **正确的GStreamer管道**

**USB摄像头（/dev/video0）**
```bash
# 错误管道
v4l2src ! video/x-raw ! ...  # 无法处理RAW10

# 正确管道
gst-launch-1.0 v4l2src device=/dev/video0 ! \
    'video/x-bayer,format=rggb10,width=1280,height=720,framerate=60/1' ! \
    videoconvert ! \
    autovideosink
```

**CSI摄像头（Jetson专用）**
```bash
gst-launch-1.0 nvarguscamerasrc ! \
    'video/x-raw(memory:NVMM),width=1280,height=720,framerate=60/1' ! \
    nvvidconv ! \
    videoconvert ! \
    autovideosink
```

#### 2. **OpenCV C++代码**

```cpp
// 关键点：必须用 appsink 而不是 autovideosink
std::string pipeline = 
    "v4l2src device=/dev/video0 ! "
    "video/x-bayer,format=rggb10,width=1280,height=720,framerate=60/1 ! "
    "videoconvert ! "
    "video/x-raw,format=BGR ! "  // OpenCV需要BGR格式
    "appsink";                    // 传递给应用程序

cv::VideoCapture cap(pipeline, cv::CAP_GSTREAMER);
```

### 四、关键知识点

#### 1. **RAW10格式特点**
- 10-bit Bayer原始数据
- 需要去马赛克（demosaic）才能变成彩色
- 常见于工业相机、高端摄像头

#### 2. **GStreamer元素区别**
| 元素            | 用途             | 输出           |
| --------------- | ---------------- | -------------- |
| `appsink`       | 传递给OpenCV应用 | 内存中的帧数据 |
| `autovideosink` | 直接显示         | 屏幕窗口       |

#### 3. **Jetson平台特殊元素**
- `nvarguscamerasrc` - CSI摄像头源
- `nvvidconv` - GPU硬件加速转换
- `(memory:NVMM)` - NVIDIA专用内存

#### 4. **Bash括号转义**
```bash
# 错误：括号被bash解释
video/x-raw(memory:NVMM)

# 正确：用引号包裹
'video/x-raw(memory:NVMM)'
# 或转义括号
video/x-raw\(memory:NVMM\)
```

### 六、完整解决方案模板

```cpp
#include <opencv2/opencv.hpp>

int main() {
    // 针对RAW10格式的完整管道
    std::string pipeline = 
        "v4l2src device=/dev/video0 ! "
        "video/x-bayer,format=rggb10,width=1280,height=720,framerate=60/1 ! "
        "videoconvert ! "
        "video/x-raw,format=BGR ! "
        "appsink";
    
    cv::VideoCapture cap(pipeline, cv::CAP_GSTREAMER);
    
    if (!cap.isOpened()) {
        std::cerr << "Failed to open camera" << std::endl;
        return -1;
    }
    
    cv::Mat frame;
    while (true) {
        cap >> frame;
        if (frame.empty()) break;
        
        cv::imshow("Camera", frame);
        if (cv::waitKey(1) == 'q') break;
    }
    
    return 0;
}
```

### 八、常见错误速查表

| 错误信息             | 原因       | 解决                               |
| -------------------- | ---------- | ---------------------------------- |
| `not-negotiated`     | 格式不匹配 | 添加 `video/x-bayer,format=rggb10` |
| `could not link`     | 元素不兼容 | 检查管道连接符 `!`                 |
| `no element "video"` | 语法错误   | 用引号包裹caps                     |
| `EGL display`        | 显示问题   | 不影响功能，可忽略                 |

**核心记忆点**：RAW10格式 → 必须用 `video/x-bayer` → 经过 `videoconvert` 转换 → 输出BGR给OpenCV → 用 `appsink` 接收。.engine格式必须在相同的环境下转换。
