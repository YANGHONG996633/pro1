# 工业视觉检测系统 - 完整项目方案

---

## 一、项目概述

**项目名称**：基于深度学习的工业产品缺陷检测系统

**应用场景**：PCB板/金属件/包装品等表面缺陷检测

**核心功能**：
- 实时图像采集与缺陷检测
- 多类型缺陷分类（划痕、污点、变形、缺失等）
- 不良品自动剔除
- 数据统计与可视化

## 五、核心功能详细设计

### 5.1 检测流程

```
光电传感器触发 → STM32发送信号 → 上位机采集图像 
    → 预处理 → 模型推理 → 判定结果 
    → 发送指令给STM32 → 执行剔除/放行
```

### 5.2 视觉算法方案

| 方案 | 模型 | 优点 | 适用场景 |
|-----|------|------|---------|
| 目标检测 | YOLOv8 | 速度快，定位准 | 明显缺陷 |
| 语义分割 | U-Net | 像素级检测 | 微小缺陷 |
| 异常检测 | PatchCore/AnoGAN | 无需大量缺陷样本 | 缺陷样本少 |

**推荐**：YOLOv8 + 传统图像处理结合

### 5.3 通信协议设计

**上位机 → STM32 指令**：
```
帧头(2B) + 命令(1B) + 数据(nB) + 校验(1B)
0xAA 0x55 + CMD + DATA + CRC8

CMD定义：
0x01: 启动传送带
0x02: 停止传送带
0x03: 设置速度
0x04: 执行剔除
0x05: 光源亮度
0x06: 心跳
```

**STM32 → 上位机 状态**：
```
0x10: 产品到位信号
0x11: 剔除完成
0x12: 状态心跳
0xFF: 错误
```

### 5.4 Qt界面功能

- 实时图像显示（原图 + 检测结果叠加）
- 检测结果列表（时间、缺陷类型、图片）
- 统计图表（良率、缺陷分布饼图）
- 参数配置（相机参数、检测阈值、通信设置）
- 历史记录查询

## 十、需要注意的问题

1. **实时性挑战**：
   - 光电触发 → 采图 → 推理 → 剔除 整个链路延迟需要精确计算
   - 如果传送带速度较快，可能需要考虑延时补偿或提前触发

2. **计算平台配置**：
   - 开发环境：Linux PC（Ubuntu 20.04/22.04），Intel i5以上 + NVIDIA独显（支持CUDA）
   - 推理加速：TensorRT将ONNX模型转为.engine，推理延迟可降至毫秒级
   - 移植方案：C++代码 + TensorRT API在Jetson平台（Orin Nano等）上可直接编译运行
   - 如无NVIDIA显卡，可先用OpenCV DNN模块进行CPU推理调试

3. **相机选型**：
   - 600万像素对于 PCB 微小缺陷检测可能不够，需根据实际检测精度需求选择
   - 低成本方案可考虑 USB 工业相机（如迈德威视）

4. **异常处理设计**：
   - 相机掉线、通信中断、气源故障等异常情况的处理
   - 急停逻辑设计

5. **数据标注工作量**：
   - 深度学习方案需要大量标注数据，这部分工作量容易被低估
   - 可考虑半监督学习或异常检测方案减少标注需求

6. **光源选型**：
   - 方案中只提到"环形光/条形光"，实际需要根据被检物体材质、缺陷类型做打光实验

7. **机械结构**：
   - 可以简化，使用现成的小型传送带模组降低开发难度
